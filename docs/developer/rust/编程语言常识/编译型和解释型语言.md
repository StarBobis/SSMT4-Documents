## 基本定义

- 编译型语言: 源代码文件将通过一套完整的编译管线, 包括宏展开器, 变量名独立化, 编译器, 汇编器, 链接器等, 得到一个可执行的二进制程序并分发给用户, 用户无需拥有完整独立的一套运行时环境.
- 解释型语言: 源代码文件直接分发给用户, 用户需要额外安装完整独立的一套运行时环境甚至是编译环境才能运行相应项目.

除此之外还有各沾一点的:

- 中间类型语言: 源代码文件将通过不完整的编译管线编译为中间文件, 有的为字节码文件, 有的则为 `.exe` 可执行文件 (于 Windows), 而将中间文件分发给用户后, 用户需要下载专门的运行时环境才能运行.

## 编译型语言介绍

### C/C++

C 语言和 C++ 都是典型的编译型语言, 源代码需要通过编译器一次翻译为目标机器的机器码, 生成独立可执行文件, 而运行时无需再次翻译.

#### 编译阶段

假设我们有源码

```c
// example.c
#include <stdio.h>
#define NUM 100
int main() {
    printf("%d", NUM);
}
```

接下来以 gcc 为例介绍编译流程.

通常采用以下指令可以完成下述所有编译阶段, 用户只会得到最终的可执行文件:

```shell
gcc example.c -o example
```

但正如我们即将介绍的, gcc 提供可选停止参数, 允许我们从任意一步开始停止到指定一步.

##### 1. 预处理器 Preprocesser

```shell
gcc -E example.c -o example.i
```

这一阶段将:

- 处理 `#include` 头文件展开;
- 处理 `#define` 宏规则;
- 处理条件编译如 `#ifdef` 等规则;
- 删除注释.

```asm
.file	"example.c"        ; 指定原始源文件名
	.text                  ; 代码段开始
	.def	printf;	.scl	3;	.type	32;	.endef
                          ; 定义 printf 符号，存储类 3（外部），类型 32（函数）
	.seh_proc	printf    ; 开始结构化异常处理过程（x64 Windows）
printf:
	pushq	%rbp          ; 保存基址指针
	.seh_pushreg	%rbp  ; SEH 信息：压入 rbp 寄存器
	pushq	%rbx          ; 保存 rbx 寄存器
	.seh_pushreg	%rbx  ; SEH 信息：压入 rbx 寄存器
	subq	$56, %rsp     ; 栈顶下移 56 字节，分配局部变量空间
	.seh_stackalloc	56    ; SEH信息：栈分配 56 字节
	leaq	48(%rsp), %rbp ; 设置帧指针，rsp+48 -> rbp
	.seh_setframe	%rbp, 48 ; SEH 信息：设置帧指针 rbp，偏移 48
	.seh_endprologue      ; SEH 序言结束
	movq	%rcx, 32(%rbp) ; 保存第一个参数（格式字符串）到栈上
	movq	%rdx, 40(%rbp) ; 保存第二个参数
	movq	%r8, 48(%rbp)  ; 保存第三个参数
	movq	%r9, 56(%rbp)  ; 保存第四个参数
	leaq	40(%rbp), %rax ; 取第二个参数的地址（可变参数列表起始地址）
	movq	%rax, -16(%rbp); 保存到局部变量
	movq	-16(%rbp), %rbx ; 加载到 rbx
	movl	$1, %ecx       ; 参数1：文件流编号 1（stdout）
	movq	__imp___acrt_iob_func(%rip), %rax ; 获取_iob_func函数地址
	call	*%rax         ; 调用 _iob_func 获取 stdout 的FILE结构指针
	movq	%rax, %rcx    ; 返回值（FILE*）作为第一个参数
	movq	32(%rbp), %rax ; 加载原始格式字符串
	movq	%rbx, %r8     ; 参数3：可变参数列表指针
	movq	%rax, %rdx    ; 参数2：格式字符串
	call	__mingw_vfprintf ; 调用 mingw 的 vfprintf 实现
	movl	%eax, -4(%rbp) ; 保存返回值（打印字符数）
	movl	-4(%rbp), %eax ; 恢复返回值到 eax
	addq	$56, %rsp     ; 释放栈空间
	popq	%rbx          ; 恢复 rbx
	popq	%rbp          ; 恢复 rbp
	ret                   ; 返回
	.seh_endproc          ; SEH 过程结束
	.def	__main;	.scl	2;	.type	32;	.endef
                          ; 定义 __main 符号，存储类 2（内部），类型 32
	.section .rdata,"dr"  ; 只读数据段
.LC0:
	.ascii "%d\0"         ; 字符串常量 "%d"
	.text                 ; 代码段
	.globl	main          ; 导出 main 符号为全局
	.def	main;	.scl	2;	.type	32;	.endef
                          ; 定义 main 符号
	.seh_proc	main      ; SEH 过程开始
main:
	pushq	%rbp          ; 保存调用者的 rbp
	.seh_pushreg	%rbp  ; SEH 寄存器信息
	movq	%rsp, %rbp    ; 设置帧指针
	.seh_setframe	%rbp, 0 ; SEH 帧指针，偏移 0
	subq	$32, %rsp     ; 分配 32 字节栈空间
	.seh_stackalloc	32    ; SEH 栈分配
	.seh_endprologue      ; SEH 序言结束
	call	__main        ; 调用 MinGW 运行时初始化
	movl	$100, %edx    ; 参数 2：值 100
	leaq	.LC0(%rip), %rax ; 加载格式串地址
	movq	%rax, %rcx    ; 参数 1：格式串
	call	printf        ; 调用 printf
	movl	$0, %eax      ; 返回值 0
	addq	$32, %rsp     ; 释放栈
	popq	%rbp          ; 恢复 rbp
	ret                   ; 返回
	.seh_endproc          ; SEH 过程结束
	.ident	"GCC: (x86_64-win32-seh-rev1, Built by MinGW-Builds project) 13.2.0" ; 编译器标识
	.def	__mingw_vfprintf;	.scl	2;	.type	32;	.endef
                          ; 外部符号声明
```

汇编码将用于一对一描述 CPU 需要做的事情, 而一款 CPU 能<ruby>做哪些事情<rt>指令集</rt></ruby>, 恰恰决定于前面所说的**指令集架构**.

##### 3. 汇编器

```shell
gcc -c example.s -o example.o # 生成可执行文件, 在 Windows 中携带后缀名 .exe
```

此时:

- 汇编代码转为**机器指令**;
- 生成目标文件;
- 不解决跨文件符号引用.

尽管汇编码是机器语言的 $1:1$ 描述, 但机器不认识字符串, 只认识 `0` 和 `1`, 须要通过一个[双射](/placeholder.md)将汇编码映射为二进制字节, 此时方能喂给 CPU, 使其如期工作.

以下以十六进制形式展示二进制文件 `example.o` 的内容.
:::tip 十六进制
你知道吗: $2$ 位十六进制数恰好对应 $1$ 个字节, 也就是 $8$ 个比特位!

分别以下标形式的 $\mathrm{Dec},\mathrm{Hex},\mathrm{Bin}$ 表示十进制 (Decimal) 数, 十六进制数 (Hexadecimal) 和二进制 (Binary), 以 $ \underline{a_1\dots a_n}_k$ 表示 $k$ 进制 $n$ 位数的直接表示, 例如 $67_\mathrm{Dec}=\underline{01000011}_\mathrm{Bin} $.

任取二进制字节 $\displaystyle a=\underline{a_1a_2a_3a_4a_5a_6a_7a_8}_\mathrm{Bin} := \left(\sum_{i=1}^8  10^{8-i}a_i\right)_\mathrm{Dec}, \forall i, a_i\in\{0,1\} $,  只需十六进制数字 $\displaystyle \underline{A_{i}}_\mathrm{Hex} = \left(\sum_{j=4i-1}^{4i+3}10^{8-j}a_j\right)_\mathrm{Dec} $, 即得 $ \displaystyle a=\underline{a_1a_2a_3a_4a_5a_6a_7a_8}_\mathrm{Bin} = \underline{A_1A_2}_\mathrm{Hex} $. 这就得到了 $1:4$ 的对应, <ruby>四个比特<rt>$ 2\times 2\times 2\times 2$</rt></ruby>恰对应一个<ruby>十六<rt>$16=2^{4}$</rt></ruby>进制数.
:::

##### 4. 链接器
```shell
gcc example.o -o example
```
这一步将会完成最后的任务:
- 符号解析: 重关联函数调用与定义;
- 重定位: 合并多个目标文件, 并分配内存地址;
- 分情况实现静态链接或动态链接.

#### 优势
理清整个编译流程就能明白编译型语言的优势何在: 用户拿到的东西将直接喂给 CPU, 没有中间商做差价, 从而得到了极致的性能, 也能够向操作系统获取更高的权限, 完成更底层的工作.

#### 劣势
如果动手试过, 你肯定理解为什么说调试周期长会是缺点, 相比解释型语言的随意, 编译型语言往往更古板一些.
