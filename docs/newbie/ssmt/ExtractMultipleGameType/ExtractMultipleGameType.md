# 提取模型有多个数据类型

SSMT的提取模型方法是遍历识别每一种可能的数据类型，所以有时候可能会错误的识别到多种数据类型。


以GF2举例，提取模型后，直接导入发现模型炸了，有的时候也表现为UV炸了：

![alt text](image.png)

此时我们可以点击数据类型右侧的按钮，进行一键重新导入另一个正确的数据类型：

![alt text](image-1.png)

![alt text](image-2.png)

顾名思义，修复该DrawIB数据类型会统计所选物体的所有DrawIB，并将属于该DrawIB的模型全部重新导入


修复该Submesh数据类型则只会统计当前所选物体Submesh

此时我们的游戏是GF2，则整个DrawIB共享相同的数据类型（终末地是每个Submesh独立数据类型）

所以我们选中一个或多个物体，点击【修复该DrawIB数据类型】，然后可以看到重新导入了：

![alt text](image-3.png)

然后虽然模型没问题了，但是要检查UV是否正确：

![alt text](image-4.png)

发现UV有问题，再点一次【修复该DrawIB数据类型】：

![alt text](image-5.png)

发现这是最后一个数据类型了，但是你发现模型仍然有问题

## 不知道DrawIB和Submesh是什么意思？

DrawIB指的是我们小键盘7 和 8选择后消失的那一块儿模型

Draw的含义是绘制，IB的含义是IndexBuffer，所以综合含义是`用于绘制的索引缓冲区`

每个 `DrawIB(用于绘制的索引缓冲区)`中都有一个或多个`Submesh`，Submesh的含义是子网格

那么连在一起就是，每一次Draw，都会绘制1到多个子网格，体现在我们提取出来的模型，有时候每个DrawIB有多个Submesh，有时候只有一个

如果选中一个模型，点击修复DrawIB的话，那就把这个模型所代表的DrawIB的所有Submesh都给修了，如果点修复Submesh的话，就只修当前选中Submesh的数据类型

这个修复并非修复，而是将当前错误的数据类型的对象，从Blender以及本地文件中都删掉，然后再自动导入一个其它数据类型，多尝试几次之后发现模型没问题了，所以看起来像是修复了，其实就是省去了我们手动去删除的一个步骤（之前的版本中都是手动去工作空间里删错误的数据类型的，现在有这俩功能方便了）

这俩按钮平时不会显示的，只有选中物体后才会显示。
